# Workflow: Add/Reuse Label

## Steps

1. **Identify the issue set and intended label.** Do not infer a new taxonomy from a single request.
2. **List existing team labels** for `CUP` and search for an exact or clearly equivalent label before creating anything.
3. **Reuse an existing label** when one matches. Create a new label only when the user explicitly approves a genuinely missing category, with a clear name, description, and color.
4. **Apply labels minimally.** Preserve existing labels unless the user explicitly requests replacement or cleanup. Use the issue update operation carefully because full label replacement can remove labels not included.
5. **Re-fetch and verify** the exact label set after the update.

## Cups & Cakes label convention

The standard coarse labels are `Feature`, `Bug`, `Chore`, `Docs`, `Content`, and `Design`. Reuse these before proposing another label.
